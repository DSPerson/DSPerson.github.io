Git 基本操作学习

发表于 2016-12-22 |

坚持真的很重要 仅此而已

从创建仓库到推送

克隆分支

git clone <url>

将代码中所需要的添加进去(一般情况下都添加 . 即所有的意思)

git add . ||(||或者) git add -A

提交

git commit -m "<1>"

备注 一：此处 引号<>中最好写东西 不写或者 引号内为空 都会上传失败不明白原因

代码拉取和推送

分两种情况

第一种 是 没有创建过远程分支 ，简单的操作方式

第二种 是 创建了远程分支，这种可以替代第一种 如果没创建远程分支稍显麻烦

第一种

git pull  直接拉取 默认的是 master
git push  直接推送 默认的是 master

第二种

git pull origin <本地分支名字>:<远程分支名字>

备注 二: origin 一般为这个名字 如果本地分支是跟踪远程分支的话可以不写远程分支 直接使用 ：git pull origin <本地分支> 就可以 一般 本地分支和远程分支名字相等

git push origin <本地分支名字>:<远程分支名字>

类比备注 二

分支操作

查看本地分支

git branch

查看所有分支 包括远程分支

git branch -a

更新分支(比如说你的同事创建了新的分支你却看不到)

git fetch

创建本地分支

git branch -b <分支名字>

创建远程分支

第一步先创建一个本地分支

git branch -b <分支名字>

第二步创建远程分支

git push origin <刚才第一步创建分支的名字>:<最好等于刚才那个分支(远程分支名字)>

代码回滚

保证当前工作区域干净(即代码没有要提交的了并且和远程分支代码一致)

可以将代码提交到远程分支上保证一致性(重复 commit –> pull –> push ) 再 继续

查看日志

git reflog

找到所需要回滚的版本

比如说 HEAD@{2} 这个是咱们回滚到的版本

git reset --hard HEAD@{2}

删除远程分支

git push origin :<远程分支名字> 
或者使用 git push origin —delete <远程分支名字>

重新创建远程分支

git push origin <远程分支名字>

创建新的远程分支 代码就变成了你想要回滚的分支中了
